# Two practical Codex workflows

This static website is Poming Zhang's approximately five-minute asynchronous contribution for multidisciplinary HDR candidates. It covers two focused practices: working with a permitted local TeX Live installation to edit and compile LaTeX through Codex, and using a reusable offline HTML briefing Skill for supervisor meetings.

This is a personal HDR contribution, not an official UWA or OpenAI guide. Tool availability varies according to the permissions and configuration of each working environment.

## Open the site

- On the published site, open the GitHub Pages URL supplied by the author.
- From a downloaded copy, keep every file in this folder together and open `index.html` in a browser.

The site has no external runtime dependencies. Its two download links use relative paths so they work both locally and from static hosting.

## Downloadable resources

- `TeX_Live_Codex_Guide.pdf` is a concise guide to environment checks, permitted installation, compilation, log inspection and human review.
- `Supervisor_Meeting_HTML_Skill.zip` contains the reusable Skill, local builder, editable template and sanitised example for an offline supervisor-meeting briefing.

The `.nojekyll` file keeps GitHub Pages deployment simple. `404.html` provides a return link when a page cannot be found.
