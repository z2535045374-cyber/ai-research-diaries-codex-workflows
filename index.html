Workflow 01 · local document production

Using Codex to edit and compile LaTeX locally

I discovered this workflow while formatting my thesis and adapting a journal manuscript. Where the working environment permits, TeX Live can be installed so that Codex can edit, compile and inspect LaTeX documents within one workflow.

This reduced repeated manual work and made template adaptation more efficient. Availability depends on the permissions and configuration of the working environment.

Codex can assist with source editing, compilation, log inspection and technical formatting corrections. Successful compilation does not prove academic accuracy. The researcher must still check wording, citations, numerical results, interpretation and journal requirements.

Five-step edit, compile and inspect workflow

  1. PrepareOpen the approved source and template requirements.
  2. EditAsk Codex to adjust the LaTeX structure and formatting.
  3. CompileRun the document through the local TeX Live installation.
  4. InspectRead the compilation log and examine the rendered output.
  5. CorrectResolve formatting issues, compile again and verify the result.

A short prompt to begin

Edit this approved LaTeX source to match the supplied journal template, compile it locally, inspect the log, and report every change that still needs my academic check.

Workflow 02 · recurring meeting preparation

Using an HTML briefing for supervisor meetings

Rebuilding a PowerPoint presentation for every supervisor meeting can be repetitive. Codex receives recent work, research progress, unresolved questions, decisions needed and next actions, then organises the material into a browser-based HTML briefing.

The page can be opened locally, shared before the meeting and updated later. It is a practical alternative for recurring research meetings, not a universal replacement for PowerPoint.

The researcher must check the content before sharing it.

Four-step input-to-output workflow

  1. CollectGather approved updates, questions and decisions for discussion.
  2. StructurePlace each item in its correct input category.
  3. GenerateBuild one self-contained, browser-based briefing.
  4. Check and shareVerify the content, open it locally and share the complete file.

A short prompt with the eight inputs

Create one local HTML supervisor briefing using only these eight approved input categories: project title; recent progress; completed work; key findings; unresolved questions; decisions required; next actions; timeline.

The downloadable skill contains the local builder, template and a sanitised example.

How to use the Skill

  1. Download and extract the Skill.
  2. Place the Skill in the supported Skills or project-instructions location used by the coding agent.
  3. Provide the eight requested inputs: project title; recent progress; completed work; key findings; unresolved questions; decisions required; next actions; timeline.
  4. Ask the coding agent to generate and verify the offline HTML briefing.
  5. Review all content before sharing it with supervisors.

The Skill was created primarily for Codex but may be adapted for other coding agents that support reusable project instructions.

Takeaway

Both workflows use AI to reduce repetitive document work, while academic judgement and final checking remain with the researcher.
