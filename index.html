00 Field note

Introduction

I am Poming Zhang, a Master of Philosophy candidate at UWA Business School. I recently submitted my thesis for examination. My main AI tools are OpenAI’s ChatGPT and Codex, and the two examples below come from my practical experience of using Codex during thesis preparation and journal manuscript development.

Local document production

01

Edit and compile LaTeX locally

I discovered this workflow while formatting my thesis and adapting a journal manuscript. Where the working environment permits, TeX Live can be installed so that Codex can edit, compile and inspect LaTeX documents within one workflow.

This reduced repeated manual work and made template adaptation more efficient. Availability depends on the permissions and configuration of the working environment.

Codex can assist with source editing, compilation, log inspection and technical formatting corrections. Successful compilation does not prove academic accuracy. The researcher must still check wording, citations, numerical results, interpretation and journal requirements.

Five-step edit, compile and inspect workflow

  1. PrepareOpen the approved source and template requirements.
  2. EditAsk Codex to adjust the LaTeX structure and formatting.
  3. CompileRun the document through the local TeX Live installation.
  4. InspectRead the compilation log and examine the rendered output.
  5. CorrectResolve formatting issues, compile again and verify the result.

Starter prompt

A short prompt to begin

Edit this approved LaTeX source to match the supplied journal template, compile it locally, inspect the log, and report every change that still needs my academic check.

Recurring meeting preparation

02

Prepare a supervisor meeting workspace

Rebuilding a PowerPoint presentation for every supervisor meeting can be repetitive. Codex can organise approved updates, findings, questions, decisions and actions into one browser-based HTML workspace.

The workspace groups the supplied material as Overview, Progress and evidence, Discussion workspace, and Actions and timeline. Search, collapsible navigation and print controls help with longer briefings without turning the page into a tutorial.

Meeting state can record discussion status, decision status, action completion and notes. Export and import keep that state in a separate JSON file; they do not change the eight-field source content.

Four-step input-to-output workflow

  1. CollectGather approved updates, questions and decisions for discussion.
  2. StructurePlace each item in its correct input field.
  3. GenerateBuild one searchable, self-contained offline meeting workspace.
  4. Check and shareCompare every displayed item with its source field before sharing.

Sanitised interactive proof

Eight-field boundary

A short prompt with the approved inputs

Create one local HTML supervisor meeting workspace using only these eight approved input fields: project title; recent progress; completed work; key findings; unresolved questions; decisions required; next actions; timeline.

The downloadable Skill includes the standard-library builder, configurable template and a sanitised example.

Use the Skill

  1. Download and extract the Skill.
  2. Place it in the supported Skills or project-instructions location used by the coding agent.
  3. Provide the eight approved input fields.
  4. Ask the coding agent to generate and verify the offline HTML workspace.
  5. Review all content before sharing it with supervisors.

The Skill was created primarily for Codex and can be adapted for coding agents that support reusable project instructions.
