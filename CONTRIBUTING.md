# Contributing

Contributions that improve the clarity, safety, accessibility or portability of these workflows are welcome.

## Prepare a change

1. Fork the repository and create a focused branch.
2. Make the smallest coherent change and keep the two-workflow scope intact.
3. Use British English in prose, interface copy, documentation and examples.
4. Use sanitised fixtures only. Never commit confidential research notes, unpublished findings, exported meeting state, personal details, email addresses, student numbers, private paths, credentials or API keys.
5. Do not add official UWA or OpenAI logos, branding or wording that suggests endorsement or affiliation.
6. Run the unit, build and publication checks described in [CUSTOMISING.md](CUSTOMISING.md).
7. Open the affected HTML at wide and narrow viewports, check keyboard operation, and inspect any generated output field by field.
8. Submit a pull request that explains the purpose, evidence and any limitations of the change.

Do not modify the bytes of `TeX_Live_Codex_Guide.pdf` merely to update repository metadata. Do not add confidential or private quality-assurance artefacts to a pull request.
